## Branching Policy

* `master` for production code
* `feature/*` for feature development. 
* `enhance/*` for minor enchancement e.g. configuration, documentation, etc.
* `bugfix/*` for bug fixes.

## Note:

* Only use "pull request" to merge any branch into `master`.
